# ADR-001: Default Model Architecture — 3D Residual Attention U-Net

**Status:** Accepted  
**Date:** 2024-01-15  
**Decided by:** Deep Learning Engineer + Research Reviewer

---

## Context

The pipeline requires a 3D synthesis model that:
- Maps 3D MRI volumes to 3D PET volumes (full-volume regression)
- Runs on a single T4 GPU (15.9 GB VRAM) in Google Colab
- Produces publication-quality PET synthesis quality

---

## Decision

**Use 3D Residual Attention U-Net as the default model.**

---

## Alternatives Considered

### Option A — 3D Residual Attention U-Net (CHOSEN)
- **Pros:** Proven in medical image synthesis; encoder-decoder with skip connections preserves fine spatial detail; attention gates suppress background activations; residual connections stabilize training; well-understood hyperparameters
- **Cons:** Fixed spatial inductive bias; no global context beyond receptive field of conv layers; more parameters than plain U-Net

### Option B — UNETR (ViT Encoder + CNN Decoder)
- **Pros:** Global context via transformer self-attention; strong on large-scale multi-scale tasks
- **Cons:** Very high VRAM for 3D inputs; slower to train; requires more data to converge; patch tokenization loses sub-patch spatial detail

### Option C — 3D Attention U-Net (without residual blocks)
- **Pros:** Lighter; faster training
- **Cons:** Harder to train deep (vanishing gradients); lower representation capacity

---

## Consequences

- The model registry supports all alternatives via `config.model.name`.
- Users can switch to UNETR or Swin UNETR for ablation studies.
- The default `features=[32, 64, 128, 256, 512]` may need reduction for T4 GPU;
  if VRAM is exceeded, reduce to `[16, 32, 64, 128, 256]`.
